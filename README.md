# SkyFitnessPro — Дипломный проект

## 📖 Описание

Веб-приложение для фитнес-платформы, позволяющее пользователям просматривать курсы, проходить тренировки, отслеживать прогресс и управлять своим профилем. Разработано в рамках дипломного проекта.

## 🛠 Стек технологий

- Frontend: Next.js 13 (Pages Router), React 18, TypeScript
- Стилизация: CSS Modules
- Аутентификация:
    - Фитнес-API с JWT-токенами
    - Данные пользователя формируются из введённых значений (email, имя)
    - React Context (`AuthContext`) + localStorage
    - Внешний API: `https://wedev-api.sky.pro/api/fitness` (авторизация, курсы, тренировки, прогресс)
- HTTP-клиенты:
    - `axios` — для авторизации
    - `fetch` — для курсов и тренировок
- Деплой: Vercel
- Инструменты: Git, npm

## 🔐 Архитектура авторизации

### Система авторизации

Проект использует фитнес-API с JWT-токенами для авторизации.
Данные пользователя (email, имя) формируются из введённых значений при регистрации/входе.

### Процесс входа/регистрации:

1. Фитнес-API (`/api/fitness/auth/login`) — получаем JWT-токен
2. Сохраняем в localStorage: токен и данные пользователя
3. Токен автоматически добавляется к запросам через заголовок Authorization
4. Функция `saveUserSession` в `AuthContext` устраняет дублирование кода

| API                     | Назначение                               | Что возвращает                                   |
| ----------------------- | ---------------------------------------- | ------------------------------------------------ |
| Фитнес (`/api/fitness`) | Авторизация, курсы, тренировки, прогресс | JWT-токен (`eyJ...`) для авторизованных запросов |

### Базовый URL для запросов

| Тип запроса                                               | Базовый URL                             | Файл-клиент                            | Особенности                                                |
| --------------------------------------------------------- | --------------------------------------- | -------------------------------------- | ---------------------------------------------------------- |
| 📚 Все запросы (авторизация, курсы, тренировки, прогресс) | `https://wedev-api.sky.pro/api/fitness` | `libs/apiConfig.ts`, `libs/apiAuth.ts` | Использует `fetch` и `axios`, заголовок `Content-Type: ""` |

### Важное требование API

Учебный сервер не принимает заголовок `Content-Type: application/json`. Все запросы должны отправляться либо с пустым заголовком, либо без него вовсе:

```typescript
// Правильно:
headers: { "Content-Type": "" }

// Неправильно (вернёт 400 ошибку):
headers: { "Content-Type": "application/json" }
```

### Требования к паролю (фитнес-API)

Фитнес-API предъявляет строгие требования к паролю. На клиенте реализована функция validateFitnessPassword, которая проверяет пароль до отправки на сервер:

- Минимум 6 символов
- Минимум 1 заглавная буква (латиница или кириллица)
- Минимум 2 спецсимвола (например: @!#?)

Примеры:

- ❌ password123 — нет заглавной, нет спецсимволов
- ❌ Password123 — нет спецсимволов
- ✅ Password@! — соответствует всем требованиям

### Защита от повреждённых данных в localStorage

В AuthContext реализована защита от повреждённого JSON в localStorage:

```typescript
try {
    setUser(JSON.parse(fitnessUser))
} catch (err) {
    console.error("Ошибка парсинга user из localStorage:", err)
    localStorage.removeItem("fitness_token")
    localStorage.removeItem("fitness_user")
}
```

## 👤 Профиль пользователя и курсы

### Отображение добавленных курсов

Компонент MyCourses загружает курсы пользователя с сервера через GET /api/fitness/users/me и отображает их в виде карточек с прогрессом:

- Загрузка списка ID курсов из response.user.selectedCourses
- Параллельная загрузка данных каждого курса через Promise.all
- Получение прогресса через GET /users/me/progress?courseId={id}
- Визуализация прогресса — анимированная полоса с градиентом

### Интерактивные карточки курсов

На главной странице и в профиле карточки курсов имеют интерактивные кнопки:

- Главная страница: кнопка + / - для добавления/удаления курса
- Страница курса: кнопка "Добавить курс" / "Курс уже добавлен ✓"
- Сохранение состояния: при перезагрузке страницы состояние кнопки восстанавливается через проверку /users/me
- Кастомный курсор: при наведении на кнопку появляется кастомный курсор с текстом действия

### Аватары пользователей

В проекте используются статические аватары (функционал загрузки собственных фото не реализован).

Архитектурное решение: Изначально в проекте был собственный бэкенд (Node.js + MongoDB + S3) для реализации функционала, которого нет во внешнем API (включая загрузку аватаров). Однако после интеграции с внешним API `wedev-api.sky.pro`, который покрывает все основные требования ТЗ (авторизация, курсы, прогресс), бэкенд стал избыточным и был удалён для упрощения архитектуры.

Почему не реализована загрузка аватаров:

- Внешний API не предоставляет эндпоинта для загрузки файлов
- Для реализации потребовалось бы вернуть собственный бэкенд как прокси между фронтендом и облачным хранилищем (Yandex Cloud S3)
- Этот функционал не входил в техническое задание и макет

Текущая реализация: Статические картинки-заглушки:

- В хедере: `/img/icon.png` (маленький аватар 42×42)
- В профиле: `/img/avatar.png` (большой аватар 197×197)

## 🎨 Страницы авторизации и регистрации

### Архитектура модалок

Формы входа и регистрации реализованы как модальные окна поверх главной страницы , а не как отдельные маршруты. Это обеспечивает:

- Плавный UX без перезагрузки страницы
- Сохранение состояния курсов и фильтров
- Единый визуальный контекст приложения

### Механизм работы:

- Ссылки в шапке ведут на `/?modal=login` и `/?modal=register`
- Главная страница (`pages/index.tsx`) отслеживает query-параметр `modal` через `useRouter`
- При наличии параметра рендерится соответствующий компонент модалки поверх контента
- Закрытие окна (клик по затемнению или успешная авторизация) возвращает пользователя в исходное состояние через `shallow routing`

### Многоуровневая валидация

Валидация данных происходит на нескольких уровнях:

1.  Клиентская (мгновенная обратная связь)

- Пустые поля — проверка `trim()` перед отправкой
- Формат email — регулярное выражение `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- Совпадение паролей — проверка идентичности в форме регистрации
- Требования фитнес-API — проверка сложности пароля через validateFitnessPassword
- Визуальная индикация — красная рамка на невалидном поле
- Сброс ошибки — при начале ввода в подсвеченном поле
- Отключение браузерной валидации — атрибут `noValidate` на форме

2.  Серверная (безопасность и бизнес-логика)

- Проверка существования пользователя в БД
- Валидация сложности пароля
- Защита от дублирования записей

3.  Обработка "двойных" ошибок

- Если курс уже добавлен — показывается сообщение "Курс уже в вашем профиле" (не ошибка)
- Если пароль не соответствует требованиям — понятное сообщение на русском

### Безопасность обработки ошибок

Внешний учебный API (`wedev-api.sky.pro`) реализует безопасную обработку ошибок авторизации : для предотвращения перебора зарегистрированных email-адресов сервер возвращает единый статус 400 без уточнения причины.

На клиенте реализована соответствующая унифицированная обработка:

- При любой ошибке авторизации показывается сообщение: _"Неверные данные. Проверьте почту и пароль"_
- Подсвечивается поле email как наиболее вероятная причина
- Это соответствует современным стандартам безопасности (OWASP)

### Защита от повторной отправки

- Кнопка отправки блокируется через `disabled={isLoading}` во время запроса
- Предотвращает дублирование запросов к серверу
- Визуально показывает процесс загрузки ("Вход..." / "Регистрация...")

## 🏗️ Архитектурные решения

### Layout-компонент

Вместо дублирования <Header /> в каждой странице создан компонент Layout, который:

- Оборачивает контент страницы
- Принимает пропс showTitle для управления заголовком
- Принимает пропс showScrollToTop для кнопки "Наверх"
- Легко расширяется (можно добавить Footer, Sidebar)

### Кастомный курсор

- Реализован кастомный курсор, который появляется при наведении на кнопки добавления/удаления курсов:
- Двойная защита от мобильных устройств:
- JavaScript: проверка window.matchMedia("(hover: hover) and (pointer: fine)")
- CSS: медиа-запросы @media (max-width: 768px) скрывают курсор
- Текст действия меняется в зависимости от состояния ("Добавить курс" / "Удалить курс")
- Плавное следование за курсором через mousemove event
- Корректная очистка event listener в useEffect cleanup

### Модальные окна подтверждения

- Вместо стандартных браузерных confirm() и alert() используются кастомные модальные окна:
- Единый дизайн во всём приложении
- Типы модалок: confirm, success, error, info
- Автоматическое закрытие через autoClose пропс
- Кастомные иконки или дефолтные по типу
- Анимация появления через @keyframes slideIn

### Компонент ScrollToTop

Переиспользуемый компонент кнопки "Наверх":

- Появляется после прокрутки на 300px
- Плавная анимация появления
- Используется на длинных страницах (главная, курс, тренировки)

### Fallback на моковые данные

При недоступности внешнего API приложение не ломается:

- На главной показываются примеры курсов из MOCK_COURSES
- На странице курса ищется соответствующий мок-курс
- Пользователь получает Toast-уведомление об ошибке

### Нормализация URL-параметров

Для защиты от массивов в router.query используется useMemo:

```typescript
const courseId = useMemo(() => {
    if (!id) return ""
```

### Возможные проблемы и решения

1. `Error: Невалидный токен`

- Причина: В localStorage лежит старый токен
- Решение: Выполнить `localStorage.clear()` и войти заново

2. Пользователь с таким email уже существует

- Причина: Уже зарегистрирован в фитнес-API
- Решение: Войти с существующими данными

3. Кнопка `+` не меняется на `-` после F5

- Причина: Не срабатывает проверка `/users/me`
- Решение: Проверить, что токен начинается с `eyJ`

4. Пароль не принимается

- Причина: Не соответствует требованиям фитнес-API
- Решение: Пароль с заглавной буквой и 2+ спецсимволами
